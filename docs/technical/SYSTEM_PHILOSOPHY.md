## 📐 수치적 방법론

### decide() 내부 구조

**현재 구조:**
1. Utility 계산
2. Softmax 확률 계산
3. 엔트로피 계산
4. 회전 토크 생성
5. Utility 재계산 (Correction)
6. Softmax 재계산 (최종)

**물리적 해석:**
- **Explicit Euler + Correction Step**
- 연속 미분방정식이 아닌 **이산 시간 동역학**
- 인지 엔진 관점에서 매우 합리적

**왜 이렇게 설계했는가:**
- ❌ 연속 시간 해밀토니언이 아님을 명확히 함
- ✅ "동역학 엔진"이라고 표현 (정확함)
- ✅ 계산적으로 안정적이고 해석 가능

## 🧠 질환의 재정의

### 전통적 관점

**질환 = 고장 (Malfunction)**
- 뇌가 "잘못 작동"함
- 정상/비정상 이분법
- 수리/치료 필요

### Cognitive Kernel 관점

**질환 = 상태공간 상의 궤도 (Orbit in State Space)**
- ADHD: 높은 엔트로피 궤도 (발산)
- ASD: 낮은 엔트로피 궤도 (수렴)
- PTSD: 트라우마 고착 궤도
- 각 질환은 **동역학적으로 다른 궤도**

**의미:**
- 질환이 "고장"이 아니라 **다른 동역학 상태**
- 상태공간에서의 궤적(trajectory)
- 물리적으로 해석 가능

## 🌊 엔트로피와 회전장

### 핵심 메커니즘

**엔트로피 → 회전장 생성:**
```
E_n = -Σ P_n(k) ln P_n(k)  (엔트로피)
    ↓
T_n(k) = γ · (E_n / ln(N)) · cos(φ_n - ψ_k)  (회전 토크)
    ↓
U_{n,k} = U_0 + α · C_n(k) + T_n(k)  (최종 효용)
```

**물리적 의미:**
- 높은 엔트로피 → 강한 회전 → 궤도 커짐 (ADHD)
- 낮은 엔트로피 → 약한 회전 → 고착 (ASD)
- 엔트로피가 자기 자신을 만드는 피드백 루프
